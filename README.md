# sites-uedsports-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能来自不同的项目、测试环境或临时需求，统一存放于此，便于版本管理、历史追溯和静态部署。

## 项目简介

`sites-uedsports-pages` 是一个纯前端资源仓库，主要包含独立的 HTML 文件及其相关资源（如 CSS、JavaScript、图片等）。仓库不针对任何单一域名或特定网站，而是作为多个页面归档的集合点。每个页面通常具有独立的功能或内容，彼此之间没有强依赖关系。

## 目录说明

仓库根目录下按以下方式组织：

- **根目录**：存放核心或通用的 HTML 页面。
- **子目录**：根据页面用途、项目来源或功能分类创建子目录，每个子目录内包含对应的 HTML 文件及配套资源。目录名称应尽量反映内容主题，但无固定命名规则。

示例结构（仅供参考，实际以仓库内容为准）：

```
/
├── index.html
├── page-example/
│   ├── index.html
│   └── style.css
└── another-page/
    └── index.html
```

## 页面归档说明

- 每个 HTML 页面应尽量保持自包含（inline 样式或资源），减少外部依赖。
- 如需引用外部资源（如图片、字体、脚本），请确保资源文件一并提交到对应目录，或使用相对路径引用。
- 页面内容、样式、功能无统一限制，但建议遵循基本的 Web 可访问性和语义化规范。
- 仓库不提供动态后端服务，所有页面均为静态内容，通过标准 HTTP 服务即可访问。

## 维护说明

- **提交规范**：提交信息应简要描述本次新增、修改或删除的页面内容。建议使用清晰、简洁的英文或中文。
- **版本管理**：利用 Git 分支管理不同阶段的页面版本，主分支（`main` 或 `master`）保持稳定可访问状态。
- **清理**：不再需要的页面及其资源应及时移除，避免仓库臃肿。如有必要，可在提交信息中注明移除原因。
- **协作**：多人协作时请先沟通目录结构和命名方式，避免冲突。Pull Request 应附上变更说明。

## 许可证

本仓库未指定特定许可证。除非另有说明，仓库内容仅供归档和参考使用。如有特殊版权或使用要求，请参考具体文件中的声明。
